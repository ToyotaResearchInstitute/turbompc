# CUDA Backends

TurboMPC includes CUDA Foreign Function Interface (FFI) backends for the
linear algebra used inside the solver.

For installation and troubleshooting, see [README](../README.md) and [build_guide.md](build_guide.md).

## Backend Summary

`TurboMPCSolver` selects forward and backward implementations independently:

```python
from turbompc.solvers.turbompc_solver import (
    BackwardBackend,
    ForwardBackend,
    TurboMPCSolver,
)

solver = TurboMPCSolver(
    program=ocp,
    params=solver_params,
    forward_backend=ForwardBackend.ADMM_FUSED_CUDSS,
    backward_backend=BackwardBackend.DIRECT_CUDSS_FFI,
)
```

The default CUDA configuration is:

```python
forward_backend=ForwardBackend.ADMM_FUSED_CUDSS
backward_backend=BackwardBackend.DIRECT_CUDSS_FFI
```

## Forward Backends

Forward backends solve the convex QP generated by each SQP iteration.

| `ForwardBackend` | Implementation | CUDA library |
|---|---|---|
| `ADMM_JAX_LOOP_PCG` | Pure-JAX ADMM loop with PCG Schur solve | None |
| `ADMM_JAX_LOOP_PCG_FFI` | JAX ADMM loop with CUDA PCG Schur solve | `libpcg_blktridi_ffi.so` |
| `ADMM_JAX_LOOP_CUDSS_FFI` | JAX ADMM loop with CUDA cuDSS Schur solve | `libcudss_blktridi_ffi.so` |
| `ADMM_JAX_LOOP_JAX_DENSE` | JAX ADMM loop with dense `jnp.linalg.solve` | None |
| `ADMM_FUSED_PCG` | Fused CUDA ADMM+PCG kernel | `libadmm_fused_ffi.so` |
| `ADMM_FUSED_CUDSS` | CUDA ADMM loop with cuDSS inner solves | `libadmm_cudss_ffi.so` |

Use the pure-JAX backend for CPU-only installs and debugging. Use
`ADMM_FUSED_CUDSS` or `ADMM_FUSED_PCG` when the CUDA kernels are available.

## Backward Backends

Backward backends compute derivatives through the MPC solve.

| `BackwardBackend` | Implementation | CUDA library |
|---|---|---|
| `ADMM_JAX_LOOP_PCG` | Pure-JAX backward ADMM loop with PCG | None |
| `ADMM_JAX_LOOP_PCG_FFI` | Backward ADMM loop with CUDA PCG Schur solve | `libpcg_blktridi_ffi.so` |
| `ADMM_JAX_LOOP_CUDSS_FFI` | Backward ADMM loop with CUDA cuDSS Schur solve | `libcudss_blktridi_ffi.so` |
| `ADMM_JAX_LOOP_JAX_DENSE` | Backward ADMM loop with dense `jnp.linalg.solve` | None |
| `ADMM_FUSED_PCG` | Fused CUDA ADMM+PCG backward path | `libadmm_fused_ffi.so` |
| `ADMM_FUSED_CUDSS` | CUDA ADMM loop with cuDSS inner solves | `libadmm_cudss_ffi.so` |
| `DIRECT_JAX_DENSE` | Direct dense KKT solve in JAX | None |
| `DIRECT_CUDSS_FFI` | Direct sparse KKT solve with cuDSS | `libcudss_sparse_kkt_ffi.so` |

`DIRECT_JAX_DENSE` is useful for small CPU/GPU debugging and finite-difference
checks. `DIRECT_CUDSS_FFI` is the CUDA sparse KKT path used by default.

## Built Libraries

`make cuda` builds the CUDA libraries under `build/ffi/`:

| Library | Used by |
|---|---|
| `libpcg_blktridi_ffi.so` | Standalone PCG Schur solve |
| `libcudss_blktridi_ffi.so` | Standalone cuDSS Schur solve |
| `libadmm_fused_ffi.so` | Fused ADMM+PCG forward/backward backend |
| `libadmm_cudss_ffi.so` | ADMM loop with cuDSS inner solves |
| `libcudss_sparse_kkt_ffi.so` | Direct sparse KKT backward solve |
| `libblktridi_common.a` | Shared static CUDA utilities |


## Block-Tridiagonal Schur Solves

The ADMM subproblem uses a Schur-complement system

```text
S z = gamma
```

where `z` stacks state/control blocks over the horizon. Dynamics only couple
neighboring time steps, so `S` is block tridiagonal. TurboMPC stores this as a
compact array with shape `(T, n, 3 * n)`, where `n = nx + nu` and each row
contains the lower, diagonal, and upper blocks.

The standalone PCG and cuDSS FFI backends solve this compact Schur system:

```python
from turbompc.solvers.linear_systems_solvers.cudss_ffi_backend import cudss_ffi_solve
from turbompc.solvers.linear_systems_solvers.pcg_ffi_backend import pcg_ffi_solve

x, iters = pcg_ffi_solve(S, Phiinv, rhs, x0, eps=1e-8, max_iters=200)
x = cudss_ffi_solve(S, rhs)
```

Most users should access these through `TurboMPCSolver` rather than calling
them directly.
